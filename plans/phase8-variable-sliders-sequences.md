## 一、概述

### 1.1 本 Phase 交付什么

Phase 8 为 Functor 增加两个核心数学交互能力：

| 功能 | 一句话描述 | 目标用户 |
|------|-----------|----------|
| **可变参数滑块** | 函数表达式中的参数（如 `y = kx + b` 中的 `k`, `b`）自动识别为可拖拽滑块，实时改变函数图像 | 所有用户，尤其是需要探索参数对函数形态影响的学生和教师 |
| **数列与级数可视化** | 以 `n` 为自变量的离散序列点图 + 部分和累积曲线 + 极限/收敛值可视化 | 高中/大学学生，学习数列、级数、收敛性判断的用户 |

### 1.2 为什么做这两个功能

1. **参数滑块是数学可视化的核心交互** — Desmos 的滑块功能是其最受欢迎的特性之一，让用户从「看一个函数」变成「探索一类函数」。这是 Functor 从"展示工具"进化为"探索工具"的关键能力。
2. **数列与级数是高中数学的核心内容** — 国内高考数学中数列占约 12-15 分，级数是高等数学的基础。目前大多数在线数学工具对数列可视化的支持较弱（如 Desmos 需要手动构造点列），Functor 可以提供原生的一键式数列/级数可视化体验。
3. **差异化竞争力** — 这两个功能组合起来，Functor 可以覆盖从初中（一次函数参数探索）到大学（级数收敛性分析）的完整数学学习链路。

### 2.2 参数自动检测逻辑

#### 2.2.1 检测算法

当用户输入函数表达式后，MathEngine 解析表达式并执行以下步骤：

```
1. 解析表达式为 AST (Abstract Syntax Tree)
2. 提取所有变量标识符（字母序列）
3. 过滤掉已知的数学常量: π, pi, e, i, phi, tau
4. 过滤掉已知的数学函数名: sin, cos, tan, log, ln, exp, sqrt, abs, ...
5. 过滤掉当前模式的独立变量:
   - 2D 模式: 过滤 x（独立变量），y（因变量）
   - 3D 模式: 过滤 x, y（独立变量），z（因变量）
   - 序列模式: 过滤 n（独立变量）
6. 剩余变量 → 视为可调节参数
```

**示例**:

| 表达式 | 独立变量 | 过滤方式 | 检测到的参数 |
|--------|---------|---------|-------------|
| `y = k*x + b` | x | x 独立, y 因变量 | `k`, `b` |
| `y = A*sin(ω*x + φ)` | x | x 独立, y 因变量 | `A`, `ω`, `φ` |
| `z = a*sin(x) + b*cos(y)` | x, y | x,y 独立, z 因变量 | `a`, `b` |
| `a_n = a1 + (n-1)*d` | n | n 独立 | `a1`, `d` |
| `y = x^2` | x | 无剩余变量 | (无参数，不显示滑块) |

#### 2.2.2 参数命名显示

- **Latin 字母**: 直接显示字母，如 `k`, `b`, `A`
- **Greek 字母**: 如果用户在数学键盘或表达式中使用命名变量（如 `omega`, `phi`），在滑块标签中显示对应的 Greek 字母符号（ω, φ）。映射表见 2.5.2 节
- **多字母参数**: 支持 `a0`, `a1`, `theta`, `alpha` 等多字符参数名

#### 2.2.3 触发时机

- 用户输入表达式后 **500ms 防抖**执行检测
- 当表达式变化时重新检测（参数新增/删除/更名均触发重新检测）
- 检测结果更新后，滑块面板自动增删对应的滑块控件

### 2.3 滑块 UI/UX

#### 2.3.1 滑块面板位置

```
┌─────────────────────────────────────┐
│ 📐 FUNCTOR                    🌙◉○○ │  TopBar
├────────────┬────────────────────────┤
│ 函数输入    │                        │
│ ┌────────┐ │                        │
│ │y=kx+b  │ │                        │
│ └────────┘ │       绘图画布区域      │
│            │                        │
│ ┌─ 参数 ──┐│                        │
│ │ k = 2.0 ││                        │
│ │ ├═══●═══┤│                        │
│ │  -5  5  ││                        │
│ │         ││                        │
│ │ b = 3.0 ││                        │
│ │ ├══●════┤│                        │
│ │  -10 10 ││                        │
│ │  ▶ 自动 ││                        │
│ └─────────┘│                        │
│            │                        │
│ 📚预设 ⚙️设置│                       │
└────────────┴────────────────────────┘
```

**设计决策**:

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|:----:|
| 滑块固定在侧边栏底部 | 与函数输入区紧密关联，不遮挡画布 | 参数多时可能需要滚动 | **默认方案** |
| 滑块作为浮动面板 | 可拖动，灵活 | 增加 UI 复杂度，可能遮挡画布 | 备选方案 |
| 滑块嵌入函数输入行 | 节省空间 | 参数多时无法展开 | 不采用 |

**结论**: 滑块面板位于侧边栏「函数输入区」下方、「预设/设置」分区上方。参数超过 5 个时显示滚动条，最大支持 8 个参数。超过 8 个参数时折叠为「展开全部」模式。

#### 2.3.2 滑块控件设计

每个参数滑块包含以下元素：

```
┌─────────────────────────────┐
│ k = 2.0              ↺ ⟲   │  ← 标题行: 参数名 + 当前值 + 重置 + 锁定
│ ├─────●──────────────┤      │  ← 滑块轨道（拖拽手柄在当前位置）
│  -5             5           │  ← 范围标签（最小值 / 最大值）
│    步长: 0.1               │  ← 步长设置（点击展开，轻量设计）
└─────────────────────────────┘
```

**交互细节**:

- **拖拽**: 鼠标拖拽手柄或在轨道上点击跳转。拖拽时数值实时更新，画布同步重绘。触屏支持触摸拖拽（最小触控区域 44px）
- **数值输入**: 点击当前值数字 → 弹出可编辑输入框（带微调按钮 + / -），支持键盘直接输入精确值
- **精度**: 默认显示 2 位小数。整数参数（如步长=1）自动显示整数
- **重置按钮** (↺): 点击将参数重置为默认值（范围中点，或用户设定的初始值）
- **锁定按钮** (⟲): 锁定后该参数值固定，拖拽无效，防止误操作。锁定状态的滑块变暗显示
- **范围标签**: 最小值/最大值显示在轨道两端。范围可点击编辑

#### 2.3.3 参数较多时的处理

| 参数数量 | 显示方式 |
|:--------:|----------|
| 1-4 | 全部展开，正常显示 |
| 5-8 | 全部展开，滑块面板出现内部滚动条，面板最大高度 320px |
| 9+ | 仅显示前 4 个 + 「展开全部 (N 个参数)」按钮，点击展开全部（可滚动），或提示「参数过多，仅前 8 个可调节」 |

#### 2.3.4 动画播放控制

当参数有动画播放功能时，滑块底部显示播放控制栏：

```
▶ / ⏸  速度: 1.0x  ↺ 循环    [范围: -5 ~ 5]
```

| 控件 | 行为 |
|------|------|
| ▶ 播放 | 参数从最小值渐变到最大值，循环或往返 |
| ⏸ 暂停 | 停在当前位置 |
| 速度 | 0.5x / 1.0x / 2.0x / 4.0x |
| 循环模式 | 往返（最小值→最大值→最小值）或 循环（最小值→最大值→跳回最小值） |
| 多个参数 | 一次只能播放一个参数的动画。播放中的参数高亮显示 |

**技术实现**: 使用 `requestAnimationFrame` 驱动，Framer Motion 的 `animate` 控制数值变化。动画过程中禁用画布重绘的防抖优化，保证 60fps 流畅更新。

### 2.4 范围配置

#### 2.4.1 默认范围

| 参数出现位置 | 默认最小值 | 默认最大值 | 默认步长 |
|-------------|:----------:|:----------:|:--------:|
| 线性参数（k, b, m 等） | -10 | 10 | 0.1 |
| 三角参数（A, ω, φ 等） | -5 | 5 | 0.1 |
| 指数参数（a, base 等） | 0.1 | 10 | 0.1 |
| 分母参数（d, r 等） | -10 | 10 | 0.1 |
| 无上下文提示的参数 | -10 | 10 | 0.1 |

#### 2.4.2 自定义范围

用户可以通过以下方式自定义范围：

1. **双击范围标签**：最小值或最大值标签可双击进入编辑模式，输入新值后按 Enter 确认
2. **步长设置**：点击步长值弹出下拉选择：0.01 / 0.1 / 0.5 / 1 / 自定义
3. **右键菜单**：右键滑块打开上下文菜单 → 「设置范围...」 → 弹出对话框设置 min/max/step/初始值

#### 2.4.3 范围记忆

- 同一会话中，用户对某个参数的范围设置会被记忆（React state）
- 保存为预设时，滑块配置（范围、步长、初始值、锁定状态）随表达式一起保存
- 作为公开预设分享时，滑块配置一并分享

### 2.7 技术考虑

#### 2.7.1 渲染性能

- **滑块拖拽实时性**: 拖拽滑块时，画布需要以 ≥ 30fps 的帧率重绘
- **防抖策略**: 滑块停止拖拽 200ms 后执行高精度重绘；拖拽中降低采样精度（2D 曲线采样点从 500 减到 200，3D 网格从 100x100 减到 50x50）
- **3D 模式降级**: 3D 滑块拖拽时，启用 Three.js 的 `setRenderLoop` 持续渲染；停止拖拽后恢复到空闲帧率（30fps）
- **Web Worker**: 参数的表达式求值可以考虑在 Web Worker 中执行，避免阻塞主线程

#### 2.7.2 状态管理

在 `functionStore.js` (Zustand) 中扩展：

```javascript
// 每个函数条目新增:
{
  ...existingFunction,
  parameters: {
    k: { value: 1, min: -10, max: 10, step: 0.1, default: 1, locked: false },
    // ...
  },
  animatingParam: null,   // 当前正在播放动画的参数名
  animationSpeed: 1.0,
}
```

#### 2.7.3 数学键盘更新

数学键盘的「变量」行需增加 `a` `b` `c` `d` `k` `m` `n` `p` 等常用参数名按键的视觉提示（accent 色边框高亮），暗示用户可以将其作为参数使用。

### 3.2 输入与模式

#### 3.2.1 输入语法

数列使用 `n` 作为自变量（与普通函数使用 `x` 区分），用户通过模式切换或特定输入格式进入数列模式。

**推荐方案：显式模式切换**

在输入框左侧增加模式选择器：

```
[ f(x) ▼ ] [y = kx + b + ]
  ┌──────┐
  │ f(x) │ ← 普通函数模式
  │ a_n  │ ← 数列模式
  │ S_n  │ ← 部分和模式
  └──────┘
```

**备选方案：自动检测**

表达式包含 `n` 且不包含 `x` 时提示用户切换到数列模式。不推荐完全自动检测，因为 `n` 也可能作为普通参数名。

#### 3.2.2 三种子模式

| 模式 | 输入语法 | 示例 | 渲染方式 |
|:----:|---------|------|----------|
| **a_n** (数列) | `表达式` 或 `a_n = 表达式` | `1/n` | 离散点 (n=1,2,3...) |
| **S_n** (部分和) | `sum(表达式, k, 1, n)` 或 `S_n = 表达式` | `sum(1/k^2, k, 1, n)` | 累积点 + 可选连线 |
| **误差** | 由 a_n 和 S_n 自动生成 | — | 显示 S_n 与极限值的差距 |

**部分和输入格式**:

```
sum(表达式, 变量, 起始值, 终止值)
```

其中终止值为 `n` 表示计算到第 n 项的部分和。

**示例**:

| 数学表示 | Functor 输入 |
|----------|-------------|
| a_n = 1/n | `1/n` |
| a_n = 1/2^n | `1/2^n` |
| a_n = (-1)^n / n | `(-1)^n / n` |
| S_n = Σ(1/k^2, k=1..n) | `sum(1/k^2, k, 1, n)` 或 `sum(1/k^2)` |
| S_n = Σ(1/2^k, k=1..n) | `sum(1/2^k, k, 1, n)` |

#### 3.2.3 简化输入

对于最常见的部分和，支持简化语法：

| 简化语法 | 等价于 | 说明 |
|---------|--------|------|
| `sum(1/k^2)` | `sum(1/k^2, k, 1, n)` | 省略参数时默认 k=1..n |
| `sum(r^(k-1))` | `sum(r^(k-1), k, 1, n)` | 同上 |

### 3.3 数列显示模式

#### 3.3.1 离散点渲染

数列以离散点的形式在 2D 坐标系中绘制：

```
值域
↑
```

#### 3.3.5 收敛/发散指示

系统对数列的收敛性提供可视化提示：

| 情况 | 视觉提示 |
|------|----------|
| **收敛** | 极限线 + 点逐步趋近极限线 + 状态栏显示「收敛」 |
| **发散** | 无极限线 + 点持续上升/下降/震荡 + 状态栏显示「发散」 |
| **震荡** | 点上下交替 + 状态栏显示「发散（震荡）」 |
| **不确定** | 不显示极限线 + 状态栏显示「计算中...」（N 增大后重新评估） |

**收敛检测算法**: 简单实现中，计算最后 10% 的项的标准差，如果标准差 < 阈值 `0.001 * (max - min)` 则认为收敛。更精确的检测可使用 Ratio Test 或 Root Test 的数值近似。

#### 3.5.3 无穷级数处理

对于无穷级数，系统通过以下方式处理：

1. 计算 N=100 的部分和作为参考值
2. 增加 N 到 500、1000 分别计算
3. 如果 S_500 与 S_1000 的差异 < 1e-6，认为已经收敛到极限值
4. 如果差异持续增大（发散），显示「级数发散」
5. 对于几何级数 `sum(r^(k-1))`，当 |r| < 1 时直接使用公式 S = 1/(1-r)

### 3.7 技术考虑

#### 3.7.1 数学引擎扩展

MathEngine 需要扩展以支持：

1. **n 变量解析**: 将 `n` 视为整数自变量，而非普通实数变量
2. **部分和计算**: `sum(expr, var, start, end)` 语法解析与求值
3. **离散采样**: 在整数点采样，而非连续的实数区间
4. **收敛检测**: 基于数值的收敛性分析

```javascript
// MathEngine 新增方法
class MathEngine {
  // 现有方法...

  // 数列求值：在整数点 n=1..N 采样
  evaluateSequence(expr, N) {
    const results = [];
    for (let n = 1; n <= N; n++) {
      const val = this.evaluate(expr, { n });
      results.push({ n, value: val });
    }
    return results;
  }

  // 部分和计算
  evaluatePartialSum(expr, N, variable = 'k') {
    const results = [];
    let sum = 0;
    for (let n = 1; n <= N; n++) {
      sum += this.evaluate(expr, { [variable]: n });
      results.push({ n, partialSum: sum });
    }
    return results;
  }

  // 收敛检测
  detectConvergence(values, threshold = 1e-6) {
    // 分析最后 10% 的数据点的标准差
    const tail = values.slice(-Math.max(10, Math.floor(values.length * 0.1)));
    const mean = tail.reduce((a, b) => a + b, 0) / tail.length;
    const variance = tail.reduce((a, b) => a + (b - mean) ** 2, 0) / tail.length;
    const stddev = Math.sqrt(variance);
    return {
      converged: stddev < threshold * (Math.max(...values) - Math.min(...values)),
      limit: mean,
      stddev,
    };
  }
}
```

| ID | 验收标准 (EARS 格式) | 优先级 |
|----|---------------------|:------:|
| AC-SEQ-01 | When 用户选择数列模式并输入 `1/n`，the system shall 在 2D 画布上以离散点的形式渲染 n=1..50 的各项 | Must |
| AC-SEQ-02 | When 用户切换显示模式为「点 + 连线」，the system shall 在点之间绘制半透明连接线 | Should |
| AC-SEQ-03 | When 用户切换为部分和模式并输入 `sum(1/k^2)`，the system shall 渲染 S_n 的累积值曲线 | Must |
| AC-SEQ-04 | When 数列收敛（如 `1/n^2`），the system shall 自动检测收敛并在画布上绘制水平虚线标注极限值 | Must |
| AC-SEQ-05 | When 数列发散（如 `1/n`），the system shall 不绘制极限线并在状态栏显示「发散」 | Must |
| AC-SEQ-06 | When 用户调整 N 值从 50 到 500，the system shall 重新计算并渲染最多 500 个数据点，响应时间不超过 500ms | Must |
| AC-SEQ-07 | When 用户 hover 到某个数据点上，the system shall 显示 Tooltip `(n=3, a₃=0.333)` 或 `(n=3, S₃=0.833)` | Should |
| AC-SEQ-08 | When 用户选择 p-级数预设 `1/n^p` 并调节 p 的滑块，the system shall 实时更新数列图像并在超过/达到收敛阈值时更新收敛指示 | Must |
| AC-SEQ-09 | When 用户使用部分和模式查看收敛级数（如 Basel 问题），the system shall 在状态栏显示「收敛到 π²/6 ≈ 1.6449」 | Should |
| AC-SEQ-10 | When 用户切换到数列模式但未输入表达式，the system shall 显示输入占位提示「输入数列通项公式，如 1/n」 | Must |
| AC-SEQ-11 | When 用户输入非法数列表达式（如 `1/(n-1)` → n=1 时除零），the system shall 跳过该点并在状态栏提示「n=1 处未定义」 | Must |
| AC-SEQ-12 | When 用户点击「逐项播放」，the system shall 从 n=1 开始逐项添加点到画布，速度约 10 项/秒 | Could |

### 6.3 跨功能验收标准

| ID | 验收标准 | 优先级 |
|----|---------|:------:|
| AC-CROSS-01 | When 滑块功能和数列功能同时启用（如数列预设带 p 参数），the system shall 在滑块面板中显示数列参数，拖拽时数列数据点实时更新 | Must |
| AC-CROSS-02 | When 用户在 2D/3D/数列模式之间切换，the system shall 保持各模式的参数配置和显示设置 | Must |
| AC-CROSS-03 | When 用户切换语言（中/英），the system shall 更新滑块面板和数列控制面板的所有标签文本 | Must |
| AC-CROSS-04 | When 用户的浏览器不支持 WebGL，the system shall 降级到 2D + 数列模式，不加载 3D 和滑块面板中的 3D 相关控件 | Should |

```
样式规范:
```
